Define three helper functions above the component (not inside it):

formatActivity(row: ActivityRow): string
  - actor = row.team_members?.display_name ?? 'Someone'
  - attendeeName = [row.attendees?.first_name, row.attendees?.last_name].filter(Boolean).join(' ') || 'an attendee'
  - Switch on row.action:
    - 'meeting_created': if row.meetings?.scheduled_at exists, format the date with toLocaleDateString('en-GB', { day: 'numeric', month: 'short' }) and toLocaleTimeString('en-GB', { hour: '2-digit', minute: '2-digit', hour12: false }), return "${actor} scheduled a meeting with ${attendeeName} for ${dateStr} at ${timeStr}". If no scheduledAt, return "${actor} flagged ${attendeeName} as 'Want to Meet'".
    - 'status_done': return "${actor} marked their meeting with ${attendeeName} as done"
    - 'action_item_added': return "${actor} added an action item for ${attendeeName}"
    - 'action_item_completed': return "${actor} completed an action item for ${attendeeName}"
    - default: return "${actor} updated a meeting"

relativeTime(iso: string): string
  - diffMs = Date.now() - new Date(iso).getTime()
  - diffMin = Math.floor(diffMs / 60_000)
  - < 1 min: 'just now'
  - < 60 min: "${diffMin}m ago"
  - < 24h: "${Math.floor(diffMin / 60)}h ago"
  - else: toLocaleDateString('en-GB', { day: 'numeric', month: 'short' })

dayLabel(iso: string): string
  - Compare d.toDateString() to today's and yesterday's toDateString()
  - Return 'Today', 'Yesterday', or toLocaleDateString('en-GB', { day: 'numeric', month: 'long' })

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase Realtime -> browser | Postgres change payloads flow from Supabase to the client WebSocket |
| browser client -> /feed (server) | router.refresh() triggers a new server render; Supabase RLS governs what data is returned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | feed/page.tsx — activity SELECT | mitigate | RLS on activity table requires auth.uid() IS NOT NULL; all four team members are trusted internal users (@ceealar.org domain). No attendee PII beyond name is exposed in feed strings. |
| T-06-02 | Denial of Service | RealtimeProvider — router.refresh() on every event | accept | Team size is 4; meetings and action items change infrequently during a 3-day conference. Supabase Realtime delivers one event per DB change; refresh thrash is not a practical risk at this scale. |
| T-06-03 | Spoofing | activity INSERT (actor_id) | accept | RLS allows any authenticated user to INSERT; actor_id is set by the application, not the DB. Trust boundary is enforced by auth session — only team members can authenticate. Server-side writes (Phase 3) set actor_id from supabase.auth.getUser(), not from client input. |
</threat_model>
