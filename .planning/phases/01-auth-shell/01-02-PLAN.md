<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server layout → Supabase | getUser() called server-side; session cookie read via Next.js cookies() |
| Client nav components → Supabase | signOut() called from browser client; no sensitive data exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Spoofing | app/(app)/layout.tsx | mitigate | Server-side getUser() check in layout; any unauthenticated request to (app) routes is redirected before rendering — middleware already redirects, layout provides defense in depth |
| T-01-06 | Information Disclosure | components/nav/profile-dropdown.tsx | accept | display_name, email, avatar_url shown to authenticated user only (layout guards access); these are the user's own data |
| T-01-07 | Tampering | Sign-out flow | mitigate | signOut() called via Supabase client which invalidates the server-side session; router.push ensures clean redirect |
</threat_model>
