    --- FILE 1: ceealar-pulse/app/api/tags/route.ts ---

    Export GET and POST:

    GET handler:
    1. createClient() from @/lib/supabase/server; call auth.getUser(); return 401 if no user.
    2. Fetch: const { data, error } = await supabase.from('tags').select('*').order('name')
    3. If error: return NextResponse.json({ error: error.message }, { status: 500 })
    4. Return NextResponse.json(data ?? [])

    POST handler:
    1. createClient(); auth.getUser(); 401 if no user.
    2. Parse body: const { name, color } = await req.json()
    3. Validate: if (!name?.trim()) return NextResponse.json({ error: 'name required' }, { status: 400 })
    4. Admin client insert:
       const admin = createAdminClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.SUPABASE_SERVICE_ROLE_KEY!, { auth: { persistSession: false } })
       const { data, error } = await admin.from('tags').insert({ name: name.trim(), color: color || '#0F766E', created_by: user.id }).select('*').single()
    5. If error: if error.code === '23505' (unique violation) return 409 with { error: 'Tag name already exists' }; else return 500.
    6. Return NextResponse.json(data, { status: 201 })

    --- FILE 2: ceealar-pulse/app/api/tags/[id]/route.ts ---

    Export PATCH and DELETE. The [id] route segment is the tag UUID.

    PATCH handler:
    1. Auth check (createClient + getUser + 401).
    2. Extract id from params: const { id } = await params (Next.js 15 async params pattern).
    3. Parse body: const { name, color } = await req.json()
    4. Fetch the tag to check ownership + system status:
       const { data: tag } = await supabase.from('tags').select('created_by, is_system').eq('id', id).single()
    5. If !tag: return 404
    6. If tag.is_system: return NextResponse.json({ error: 'System tags cannot be modified' }, { status: 403 })
    7. If tag.created_by !== user.id: return NextResponse.json({ error: 'Not your tag' }, { status: 403 })
    8. Build update object: const update: Record<string, string> = {}; if (name?.trim()) update.name = name.trim(); if (color) update.color = color
    9. Admin client update:
       const { data, error } = await admin.from('tags').update(update).eq('id', id).select('*').single()
    10. If error: if 23505 return 409; else 500.
    11. Return NextResponse.json(data)

    DELETE handler:
    1. Auth check.
    2. Extract id from params.
    3. Fetch tag: check created_by, is_system.
    4. If !tag: 404. If is_system: 403 "System tags cannot be deleted". If tag.created_by !== user.id: 403.
    5. Check for existing assignments:
       const { count } = await supabase.from('attendee_tags').select('*', { count: 'exact', head: true }).eq('tag_id', id)
       if (count && count > 0) return NextResponse.json({ error: 'Tag is in use — remove all assignments first' }, { status: 409 })
    6. Admin delete: await admin.from('tags').delete().eq('id', id)
    7. Return NextResponse.json({}, { status: 204 }) — note: use status 200 with empty body since 204 cannot have a body in Next.js route handlers; use: return new NextResponse(null, { status: 204 })

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /api/tags | name/color from untrusted client |
| Browser → PATCH /api/tags/[id] | tag ownership checked server-side |
| Browser → DELETE /api/tags/[id] | system-tag guard server-side; assignment count checked |
| Browser → POST /api/attendees/[id]/tags | tag_id from client — any authenticated user can assign |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | PATCH /api/tags/[id] | mitigate | Route fetches tag.created_by and checks user.id; returns 403 if mismatch; system tags additionally blocked |
| T-04-02 | Elevation of Privilege | DELETE /api/tags/[id] | mitigate | Same ownership check as PATCH; system tag guard; assignment count guard prevents orphan data |
| T-04-03 | Spoofing | created_by on new tags | mitigate | created_by is set from server-side user.id (auth.getUser()), never from request body |
| T-04-04 | Tampering | Tag name uniqueness | mitigate | Postgres unique constraint on tags.name; route returns 409 on duplicate so client can show clear error |
| T-04-05 | Information Disclosure | GET /api/tags | mitigate | Route calls auth.getUser() before returning data; 401 for unauthenticated requests |
| T-04-06 | Denial of Service | POST /api/tags (unbounded creates) | accept | 4-person internal tool; no rate limiting needed; Postgres constraint prevents exact duplicate spam |
</threat_model>
