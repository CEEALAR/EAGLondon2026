    1. meetings table:
    ```
    create table public.meetings (
      id uuid primary key default gen_random_uuid(),
      attendee_id uuid not null references public.attendees(id) on delete cascade,
      owner_id uuid not null references auth.users(id),
      status text not null default 'want_to_meet'
        check (status in ('want_to_meet','planned','done','no_show','cancelled')),
      scheduled_at timestamptz,
      duration_minutes int,
      location text,
      prep_note text,
      summary text,
      meeting_notes text,
      comments text,
      follow_up_date date,
      created_at timestamptz not null default now(),
      updated_at timestamptz not null default now()
    );
    ```
    - Enable RLS: alter table public.meetings enable row level security;
    - RLS policy SELECT (all authenticated): create policy "auth users select meetings" on public.meetings for select using (auth.uid() is not null);
    - RLS policy INSERT (all authenticated): create policy "auth users insert meetings" on public.meetings for insert with check (auth.uid() is not null);
    - RLS policy UPDATE (all authenticated): create policy "auth users update meetings" on public.meetings for update using (auth.uid() is not null);

    2. updated_at trigger for meetings (reuse the same function pattern as attendees — check if update_updated_at() already exists before creating):
    ```
    create or replace function update_updated_at()
    returns trigger language plpgsql as $$
    begin new.updated_at = now(); return new; end;
    $$;
    create trigger meetings_updated_at before update on public.meetings
      for each row execute function update_updated_at();
    ```
    Use "create or replace" so it is safe even if the function was created in migration 0002.

    3. action_items table:
    ```
    create table public.action_items (
      id uuid primary key default gen_random_uuid(),
      meeting_id uuid not null references public.meetings(id) on delete cascade,
      text text not null,
      due_date date,
      done boolean not null default false,
      created_by uuid not null references auth.users(id),
      created_at timestamptz not null default now()
    );
    ```
    - Enable RLS: alter table public.action_items enable row level security;
    - RLS SELECT (authenticated): create policy "auth users select action_items" on public.action_items for select using (auth.uid() is not null);
    - RLS INSERT (authenticated): create policy "auth users insert action_items" on public.action_items for insert with check (auth.uid() is not null);
    - RLS UPDATE (authenticated): create policy "auth users update action_items" on public.action_items for update using (auth.uid() is not null);
    - RLS DELETE (authenticated): create policy "auth users delete action_items" on public.action_items for delete using (auth.uid() is not null);

    4. activity table:
    ```
    create table public.activity (
      id uuid primary key default gen_random_uuid(),
      actor_id uuid not null references auth.users(id),
      meeting_id uuid references public.meetings(id) on delete set null,
      attendee_id uuid references public.attendees(id) on delete set null,
      action_type text not null,
      body text not null,
      created_at timestamptz not null default now()
    );
    ```
    action_type check constraint: check (action_type in ('meeting_created','status_done','action_item_added','action_item_completed'))
    - Enable RLS: alter table public.activity enable row level security;
    - RLS SELECT (authenticated): create policy "auth users select activity" on public.activity for select using (auth.uid() is not null);
    - RLS INSERT (authenticated): create policy "auth users insert activity" on public.activity for insert with check (auth.uid() is not null);

    5. meetings_view (prep_note privacy):
    ```
    create view public.meetings_view as
    select
      m.*,
      case
        when m.owner_id = auth.uid() or m.status = 'done' then m.prep_note
        else null
      end as prep_note
    from public.meetings m;
    ```
    Note: this view overrides the prep_note column from m.* with a privacy-filtered version. The SELECT * from the view returns the filtered prep_note. Do NOT add grant statements — the project uses RLS + anon key; the view inherits the caller's RLS context.

    ```typescript
    import { createClient } from '@/lib/supabase/server'
    import { TeamMember } from '@/lib/types'

    export async function fetchTeamMembers(): Promise<TeamMember[]> {
      const supabase = await createClient()
      const { data, error } = await supabase
        .from('team_members')
        .select('id, email, display_name, avatar_url')
        .order('display_name')
      if (error || !data) return []
      return data as TeamMember[]
    }
    ```

    This function is called by server components and API routes that need to populate the owner dropdown. It must NOT be imported in client components directly — pass the result as a prop.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase view → client | meetings_view CASE expression enforces prep_note privacy at the DB layer |
| auth.users → meetings.owner_id | owner_id references auth.users — only valid user UUIDs can be set |
| RLS → action_items | All authenticated users can delete any action_item — intentional for 4-person team tool |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | prep_note field | mitigate | meetings_view CASE expression returns null for non-owners when status != 'done'; all reads of prep_note must use meetings_view, never the base meetings table |
| T-03-02 | Elevation of Privilege | meetings INSERT | mitigate | RLS insert policy requires auth.uid() is not null; middleware blocks unauthenticated requests before reaching Supabase |
| T-03-03 | Tampering | activity table insert | accept | activity rows are append-only by design; no UPDATE/DELETE RLS policy on activity means rows cannot be mutated after creation — this is the desired audit log behavior |
| T-03-04 | Information Disclosure | team_members data | accept | All four team members are trusted equals; display_name and email are not sensitive in this context |
</threat_model>
