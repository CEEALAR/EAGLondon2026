  <action>
    Create GET /api/export that returns a CSV download response.

    Auth check: use createClient() from @/lib/supabase/server, call getUser(), return 401 if no session.

    Data fetch: use service role admin client (createAdminClient with NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY) to query:
    - attendees table: all columns (select *)
    - joined: attendee_tags with nested tags(name) for tag names
    - joined: meetings aggregate — count total meetings, get last meeting status, count open action items

    Because Supabase JS doesn't support aggregates in select strings, do two queries:
    1. Main: adminClient.from('attendees').select('*, attendee_tags(tag_id, tags(name))').order('last_name')
    2. Meetings summary: adminClient.from('meetings').select('attendee_id, status, action_items(done)')

    Build a map from attendee_id to { meeting_count, last_status, open_action_items } by iterating meetings:
    - meeting_count: count all meetings per attendee_id
    - last_status: status of the most recently created meeting (use array ordering from query sorted by created_at desc — add .order('created_at', { ascending: false }) to meetings query)
    - open_action_items: sum of action_items where done=false across all meetings for that attendee

    CSV headers (exact order):
    id, swapcard_url, first_name, last_name, company, job_title, career_stage, biography, expertise, interests, how_others_can_help, how_i_can_help, country, seeking_work, recruitment, linkedin, why_they_matter, how_to_engage, hypothesis, risks, collaboration_hooks, tags, meeting_count, last_meeting_status, open_action_items, created_at, updated_at

    For array fields (expertise, interests): join with semicolons.
    For tags: join tag names with commas.
    For boolean fields (seeking_work): output "true"/"false"/"" (empty for null).
    CSV escape rule: if a cell value contains comma, double-quote, or newline, wrap in double-quotes and escape internal double-quotes by doubling them ("").

    Build the CSV string in memory (1904 rows is ~500KB, fine for Node).

    Return: new Response(csvString, { headers: { 'Content-Type': 'text/csv', 'Content-Disposition': 'attachment; filename="ceealar-pulse-export.csv"' } })
  </action>
