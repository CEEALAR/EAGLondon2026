**As a** CEEALAR team member, **I want to** receive a browser notification 10 minutes before each of my scheduled meetings, **so that** I'm never caught off-guard during the conference.

Purpose: Conference-day safety net so meetings don't get missed.
Output: NotificationProvider component wired into app layout with full permission + scheduling logic.
</objective>

    Create a 'use client' component NotificationProvider that wraps children and manages notification scheduling.

    Props: { children: React.ReactNode }

    Logic in a single useEffect with a router dependency (import useRouter from next/navigation):

    1. Permission request: if Notification.permission === 'default', call Notification.requestPermission() — do NOT await it blocking; fire-and-forget via .then().

    2. Schedule function (scheduleNotifications): called on mount and on router change:
       a. Fetch today's meetings: use createClient() from @/lib/supabase/client.
          Query: from('meetings').select('id, scheduled_at, attendees(first_name, last_name)').not('scheduled_at', 'is', null).eq('status', 'planned').gte('scheduled_at', todayStart).lte('scheduled_at', todayEnd)
          where todayStart = new Date() with hours/min/sec set to 0, todayEnd = same day 23:59:59 (use ISO strings).
       b. For each meeting row: compute msUntil10MinBefore = new Date(scheduled_at).getTime() - Date.now() - 10 * 60 * 1000
       c. If msUntil10MinBefore > 0: call setTimeout(() => { new Notification('Meeting in 10 minutes', { body: 'Meeting with ' + attendeeName + ' at ' + timeStr, icon: '/apple-touch-icon.png' }) }, msUntil10MinBefore). Collect all timer IDs.
       d. Return cleanup: a function that calls clearTimeout on all collected IDs.

    3. The useEffect dependency array should include router (so it re-runs on each router.refresh(), which RealtimeProvider triggers). Actually router reference is stable — better approach: use a key or router.events. The simplest approach that works with Next.js 15 App Router: use a refresh counter state that RealtimeProvider does NOT control. Instead, add a pathname-based dependency: import usePathname from next/navigation; include pathname in deps — this fires on navigation but NOT on router.refresh().

    Better approach: use a dedicated useEffect that subscribes to the same Supabase channel independently. On any INSERT/UPDATE on meetings: call scheduleNotifications(). This is fully self-contained.

    Final design:
    - useEffect 1: request permission on mount (runs once)
    - useEffect 2: subscribe to Supabase Realtime on meetings table (INSERT/UPDATE/DELETE) — on any event, call scheduleNotifications(). Also call scheduleNotifications() immediately on mount. Cleanup: clearTimeout all timers + removeChannel.
    - scheduleNotifications is a stable function (useCallback with no deps, or plain function defined inside effect).
    - Guard: if (Notification.permission !== 'granted') return early from scheduleNotifications.
    - Guard: if (typeof Notification === 'undefined') return early — for SSR safety.

    For attendee name in notification body: the Supabase join returns attendees as an object or null. Use optional chaining: meeting.attendees?.first_name + ' ' + meeting.attendees?.last_name, trim, fallback to 'an attendee'.

    The time string: new Date(scheduled_at).toLocaleTimeString('en-GB', { hour: '2-digit', minute: '2-digit', hour12: false }).

    TypeScript: the meetings query result type — cast to a local type:
    type MeetingForNotif = { id: string; scheduled_at: string; attendees: { first_name: string | null; last_name: string | null } | null }

    Do NOT use `any` types. Use `as unknown as MeetingForNotif[]` for the Supabase result cast.

    Return JSX: <>{children}</>
